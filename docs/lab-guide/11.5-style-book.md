# Lab 11.5 â€” Build and Apply a Liferay Style Book

## Overview

Expose the Nexcent design system as editable Liferay frontend tokens, create a Style Book in the Liferay UI, and apply it to the landing page without rebuilding the React Custom Elements.

At the end of this lab:

- A `themeCSS` Client Extension provides a frontend token definition.
- The Style Book editor exposes Nexcent colors, typography, container width, spacing, and radius values.
- Global CSS consumes Style Book values through stable CSS-variable aliases.
- A content editor can update the page appearance without changing source code.
- The responsibilities of Theme CSS, Style Book, Global CSS, and component CSS are clearly separated.

## Step 2 â€” Configure the Theme CSS Client Extension

Open `client-extension.yaml`:

```yaml
nexcent-theme-css:
    clayURL: css/clay.css
    frontendTokenDefinitionJSON: src/frontend-token-definition.json
    mainURL: css/main.css
    name: Nexcent Theme CSS
    type: themeCSS
```

`clayURL` and `mainURL` point to the CSS generated from the `styled` design pack. `frontendTokenDefinitionJSON` registers the editable tokens used by Style Books.

Do not set `scope: controlPanel`. This Theme CSS is intended for site pages, not administrative pages.

## Step 3 â€” Understand the frontend token definition

Open `src/frontend-token-definition.json`.

Each token has:

- A stable `name` used by Liferay.
- A label shown to editors.
- A default value.
- An optional editor type or list of valid values.
- A mapping to a CSS variable.

Example color token:

```json
{
  "defaultValue": "#4caf4f",
  "editorType": "ColorPicker",
  "label": "Primary Color",
  "mappings": [
    {
      "type": "cssVariable",
      "value": "nxc-style-primary"
    }
  ],
  "name": "nexcentPrimaryColor",
  "type": "String"
}
```

The mapping value `nxc-style-primary` becomes the CSS variable:

```css
--nxc-style-primary
```

The sample definition exposes two categories:

```text
Nexcent Brand
â”śâ”€â”€ Colors
â””â”€â”€ Typography

Nexcent Layout
â””â”€â”€ Layout and Shape
```

Editable values include:

- Primary and hover colors.
- Heading, body, muted, and surface colors.
- Font family and section heading size.
- Container width.
- Section vertical spacing.
- Button and card radius.

## Step 4 â€” Connect Style Book tokens to Global CSS

Open:

```text
client-extensions/nexcent-global-assets/assets/global.css
```

Global CSS exposes stable variables used by all components:

```css
:root {
    --nxc-color-primary: var(--nxc-style-primary, #4caf4f);
    --nxc-color-primary-hover: var(--nxc-style-primary-hover, #388e3b);
    --nxc-color-heading: var(--nxc-style-heading, #263238);
    --nxc-color-text: var(--nxc-style-text, #4d4d4d);
    --nxc-color-muted: var(--nxc-style-muted, #717171);
    --nxc-color-surface: var(--nxc-style-surface, #f5f7fa);
    --nxc-font-family: var(
        --nxc-style-font-family,
        Inter,
        system-ui,
        sans-serif
    );
    --nxc-container-width: var(--nxc-style-container-width, 72rem);
    --nxc-section-space: var(--nxc-style-section-space, 4rem);
    --nxc-radius-sm: var(--nxc-style-radius-sm, 0.25rem);
    --nxc-radius-md: var(--nxc-style-radius-md, 0.75rem);
}
```

This alias layer has two benefits:

1. Components keep using the existing `--nxc-*` contract.
2. The page still renders correctly before a Style Book is applied because every variable has a fallback.

## Responsibility matrix

| Layer | Responsibility | Examples |
|---|---|---|
| Theme CSS | Provides complete theme CSS and frontend token definitions | `clay.css`, `main.css`, `frontend-token-definition.json` |
| Style Book | Stores editor-selected values | Brand colors, font, width, spacing, radius |
| Global CSS | Provides shared runtime aliases and utilities | `.nxc-button`, focus ring, reduced motion |
| Component CSS | Controls component-specific layout | Hero grid, Services cards, Feature image order |
| Web Content | Stores business content | Titles, descriptions, images, links |

Do not put article titles, service names, API IDs, or migration settings into a Style Book.

## Troubleshooting

### Nexcent Theme CSS does not appear when creating the Style Book

1. Confirm the Theme CSS Client Extension deployed successfully.
2. Confirm `frontendTokenDefinitionJSON` points to `src/frontend-token-definition.json`.
3. Confirm the JSON contains `frontendTokenCategories`.
4. Confirm Theme Scoped Style Books is enabled when available.
5. Reapply Nexcent Theme CSS to the page set and refresh.

### The page loses Liferay widget styling

Do not deploy empty `clay.css` or `main.css` files. The project must build from the `styled` design pack so the Theme CSS contains the required base styles.

### Style Book values appear but components do not change

1. Inspect `--nxc-style-primary` in the browser.
2. Inspect `--nxc-color-primary`.
3. Confirm Global CSS uses `var(--nxc-style-primary, #4caf4f)`.
4. Confirm the component uses `var(--nxc-color-primary)` instead of a hard-coded color.
5. Clear cached Client Extension assets.

### The Style Book is incompatible with the page

Apply **Nexcent Theme CSS** to the page first, then select a Style Book created for that same Theme CSS Client Extension.
